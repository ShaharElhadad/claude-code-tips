מה זה CLAUDE.md?

קובץ אחד פשוט שחוסך לכם שעות של הסברים חוזרים.

תחשבו על זה ככה:

נכנס אליכם מפתח חדש לצוות. ביום הראשון הוא לא יודע כלום. איזה פריימוורק? איזה סגנון קוד? איפה שמים מה?

עכשיו דמיינו שנותנים לו דף אחד שמסביר הכל: מבנה הפרויקט, הכלים, הכללים, ומה אסור לשנות. פתאום הוא פרודוקטיבי מהיום הראשון.

CLAUDE.md הוא הדף הזה. Claude Code קורא אותו אוטומטית בתחילת כל שיחה, ומיד מבין את ההקשר.

הסבר

איפה שמים את הקובץ?

פשוט יוצרים קובץ בשם CLAUDE.md בתיקייה הראשית של הפרויקט. Claude Code מזהה אותו אוטומטית. בלי הגדרות, בלי קונפיגורציה.

אפשר גם לשים CLAUDE.md בתת תיקיות. כל תיקייה יכולה להכיל כללים ספציפיים לחלק הזה של הפרויקט.

חשוב להבין

זה לא רק לפרויקטי קוד

CLAUDE.md עובד עם כל סוג של פרויקט. אתר שיווקי, סקריפטים, תיקיית מסמכים, כל דבר. המטרה היא לתת ל-Claude Code הקשר כדי שיעבוד חכם יותר.

איך בונים CLAUDE.md מושלם

4 שלבים פשוטים שייתנו לכם קובץ CLAUDE.md שעובד.

1

הגדירו את הפרויקט

תתחילו בתיאור קצר של מה הפרויקט, באיזה טכנולוגיה, ומה המטרה.

CLAUDE.md
# My Project
This is a Next.js 14 app with TypeScript,
Tailwind CSS, and Prisma ORM.
It's a SaaS dashboard for managing
customer subscriptions.
2

כתבו את הכללים

מה הסגנון? מה אסור? איזה ספריות מותר להשתמש? הכללים האלה ימנעו טעויות חוזרות.

CLAUDE.md
## Rules
- Always use TypeScript, never JavaScript
- Use Tailwind for styling, no CSS files
- Prefer server components by default
- Never modify the database schema
without asking first
- Use date-fns, not moment.js
3

תארו את המבנה

עזרו ל-Claude Code להבין איפה מה נמצא. ככה הוא יפנה לקבצים הנכונים מההתחלה.

CLAUDE.md
## Project Structure
src/app/ → Next.js pages and routes
src/lib/ → Shared utilities
src/components/ → Reusable UI components
prisma/ → Database schema
scripts/ → Build and deploy scripts
4

הוסיפו פקודות שימושיות

תכתבו את הפקודות שאתם משתמשים בהן. ככה Claude Code יידע איך להריץ, לבדוק, ולבנות.

CLAUDE.md
## Commands
npm run dev → Start dev server
npm run build → Production build
npm test → Run all tests
npx prisma db push → Sync DB schema
טיפ: אתם יכולים לבקש מ-Claude Code עצמו לבנות לכם CLAUDE.md. פשוט כתבו: "scan this project and create a CLAUDE.md file with rules and structure"
לרמה הבאה

5 טריקים שמשדרגים את CLAUDE.md

1

CLAUDE.md בתת תיקיות

שמו CLAUDE.md בתוך src/components/ עם כללי עיצוב ספציפיים. כל תיקייה יכולה להכיל הנחיות משלה. Claude Code ממזג את כולם אוטומטית.

2

דוגמאות בתוך הכללים

במקום להגיד "use consistent naming", תראו דוגמה: "Name components like UserCard.tsx, not user_card.tsx". דוגמאות שוות יותר מהסברים.

3

עדכנו אותו תוך כדי עבודה

מצאתם ש-Claude Code עושה טעות חוזרת? תוסיפו כלל חדש ל-CLAUDE.md. ככה הטעות לא תחזור יותר.

4

הגדירו את גבולות הגזרה

כתבו בפירוש מה אסור לשנות: "Never modify files in /config, /migrations, or .env". ככה הוא לא יגע בקבצים רגישים.

5

שתפו אותו עם הצוות

CLAUDE.md הוא קובץ רגיל בפרויקט. תעשו לו commit ל-Git. ככה כל הצוות עובד עם אותם כללים ואותו הקשר.

שאלות נפוצות

שאלות ותשובות

כמה ארוך צריך להיות CLAUDE.md?
אין מגבלה, אבל פרקטית 20 עד 50 שורות מספיקות לרוב הפרויקטים. העיקר: כל שורה צריכה להוסיף מידע שימושי. עדיף קצר וממוקד על ארוך ומבולבל.
מה קורה אם אין CLAUDE.md?
Claude Code עדיין עובד, אבל הוא צריך לגלות הכל לבד. הוא יקרא קבצים, ינסה להבין את המבנה, ולפעמים יטעה. CLAUDE.md חוסך את כל ההתלבטות הזו.
עברית או אנגלית ב-CLAUDE.md?
אנגלית מומלצת כי Claude Code אופטימלי עם אנגלית טכנית. אבל אפשר בהחלט לשלב. למשל, כללים באנגלית ותיאורים בעברית. העיקר שהמידע ברור.
איך מבקשים מ-Claude Code לבנות CLAUDE.md?
פשוט תכתבו בתוך Claude Code: "Scan this project and create a comprehensive CLAUDE.md file. Include the tech stack, code style rules, project structure, and common commands." הוא ייצור את הקובץ אוטומטית.

רוצים ליצור CLAUDE.md עכשיו?

פתחו את Claude Code בפרויקט שלכם ובקשו ממנו לסרוק ולבנות את הקובץ. זה לוקח דקה אחת.
