מה זה בכלל Prompt Engineering?

בשפה פשוטה, בלי מילים מסובכות. מתאים גם למי שרק מתחיל.

תחשבו על זה ככה:

אתם נכנסים למסעדה ואומרים למלצר "תביא לי משהו טוב". מה תקבלו? מי יודע. אולי משהו שאתם אוהבים, אולי לא.

עכשיו דמיינו שאתם אומרים: "אני רוצה סטייק מדיום, עם תוספת אורז ירוק, בלי בצל." פתאום אתם מקבלים בדיוק מה שרציתם.

Prompt Engineering זה ללמוד איך להזמין. ככל שתהיו יותר ספציפיים, התוצאה תהיה יותר מדויקת.

הסבר

ב Claude Code זה אותו דבר בדיוק

אתם יכולים לכתוב "תתקן את הקוד" ולקוות לטוב. או שתכתבו בדיוק מה צריך לקרות, איפה הבעיה, ומה התוצאה הרצויה.

ההבדל בין השניים? פרומפט טוב חוסך לכם 5 סיבובים של הבהרות וחוזר עם תשובה מושלמת כבר בפעם הראשונה.

חשוב להבין

זה לא "טריקים סודיים"

Prompt Engineering זה לא מילות קסם שצריך לזכור. זה לא תבניות מסובכות שצריך להעתיק.

זה פשוט תקשורת ברורה. ככל שתהיו יותר ברורים לגבי מה אתם רוצים, Claude Code ייתן לכם תוצאות יותר טובות.

דוגמה מהחיים

פרומפט חלש מול פרומפט חזק

בואו נראה את ההבדל בפועל. אותה משימה, שתי גישות שונות לחלוטין.

פרומפט חלש
"תתקן את הבאג" — באיזה קובץ? איזה באג? מה קורה ומה היה צריך לקרות?
"תבנה לי אתר" — אין שום הקשר. איזה אתר? למי? עם מה?
"תשפר את הקוד" — לשפר מה? ביצועים? קריאות? מבנה? הכל ביחד?
תוצאות לא מדויקות, צריך עוד 5 סיבובים של הבהרות
פרומפט חזק
הקשר ברור — מציינים את הקובץ, השפה, והפריימוורק
בעיה מוגדרת — מתארים בדיוק מה קורה ומה היה צריך לקרות
תוצאה רצויה — מגדירים איך צריך להיראות הפלט הסופי
אילוצים — מה לא לשנות, אילו ספריות להשתמש, סגנון קוד
תוצאה מדויקת כבר בפעם הראשונה, בלי סיבובים מיותרים
שימו לב: פרומפט חזק לא חייב להיות ארוך. 2 עד 3 משפטים ממוקדים עדיפים על פסקה ארוכה ומבולבלת.
עכשיו לחלק הכיף

4 דפוסים שעובדים תמיד

הנה 4 תבניות פשוטות שישדרגו לכם את התוצאות באופן מיידי.

1

הקשר + בקשה

תמיד תתחילו עם הקשר. תגידו ל-Claude Code מה הפרויקט, באיזו שפה, ומה אתם מנסים להשיג.

Terminal
~ # bad: no context
~ add a login page
~ # good: with context
~ this is a Next.js 14 app with Tailwind. Add a login page at /login with email and password fields, form validation, and a submit handler that calls /api/auth/login
2

מה קורה + מה צריך לקרות

כשמדווחים על באג, תתארו את המצב הנוכחי ואת המצב הרצוי. ככה Claude Code מבין בדיוק מה לתקן.

Terminal
~ # describe current vs expected
~ in src/cart.ts, when adding the same item twice, it creates a duplicate instead of incrementing the quantity. Fix it so adding an existing item increases quantity by 1
3

תנו דוגמה

אם אתם רוצים פורמט מסוים, תראו ל-Claude Code דוגמה. תמונה שווה אלף מילים, ודוגמה שווה אלף הסברים.

Terminal
~ # show an example of what you want
~ create a TypeScript type for API responses. Follow this pattern:
type ApiResponse<T> = { success: true; data: T } | { success: false; error: string }
create similar types for PaginatedResponse and StreamResponse
4

הגדירו מה לא לעשות

לפעמים חשוב להגיד מה לא לשנות. ככה אתם מונעים שינויים לא רצויים.

Terminal
~ # set clear boundaries
~ refactor the auth module to use JWT tokens. Don't change the database schema, don't modify the API routes, only change the auth logic in src/lib/auth.ts
טיפ: אם Claude Code עשה משהו שלא רציתם, אל תתחילו מחדש. פשוט תגידו לו מה לשנות: "No, keep the original header and only change the footer"
טעויות נפוצות

5 טעויות שכולם עושים (ואיך לתקן)

1

פרומפטים כלליים מדי

"תבנה לי אפליקציה" זה כמו להגיד לאדריכל "תבנה לי בית". ספציפיות היא המפתח. תגדירו מה, איפה, ואיך.

2

יותר מדי דברים בבקשה אחת

לשלוח פרומפט עם 10 משימות שונות. עדיף לשבור ל-3 עד 4 בקשות ממוקדות. Claude Code יעבוד טוב יותר ככה.

3

לא לתאר את הבעיה

לכתוב "זה לא עובד" בלי לפרט מה קורה. תצרפו את הודעת השגיאה, את השורה הבעייתית, ומה ניסיתם.

4

לא לבדוק ולתקן

Claude Code נתן לכם תוצאה שקרובה אבל לא מושלמת? אל תתחילו מחדש. תגידו לו מה לשנות. הוא לומד מהשיחה.

5

לשכוח את CLAUDE.md

קובץ CLAUDE.md בתיקיית הפרויקט מגדיר כללים קבועים. במקום לחזור על אותן הוראות כל פעם, תכתבו אותם שם פעם אחת.

שאלות נפוצות

שאלות ותשובות

עברית או אנגלית? מה עדיף?
Claude Code מבין עברית מצוין. אבל לפרומפטים טכניים, אנגלית עובדת קצת יותר טוב כי רוב הדוקומנטציה והדוגמאות הן באנגלית. הטיפ: תשתמשו באנגלית למונחים טכניים ובשפה שנוחה לכם לתיאור.
כמה ארוך צריך להיות פרומפט?
אין כלל קשיח. משפט אחד ממוקד עדיף על פסקה ארוכה ומבולבלת. אבל אם המשימה מורכבת, תרגישו חופשי לכתוב 5 עד 10 שורות. העיקר: כל משפט צריך להוסיף מידע חדש, בלי חזרות מיותרות.
מה עושים כשהתוצאה לא מה שרציתי?
אל תתחילו שיחה חדשה. פשוט תגידו מה לא בסדר: "The header looks good but change the font size to 24px and make the background darker." Claude Code שומר על ההקשר ויתקן בהתאם. חשבו על זה כמו שיחה עם קולגה.
האם CLAUDE.md באמת משנה?
בהחלט. CLAUDE.md הוא כמו brief שאתם נותנים למפתח חדש. הוא מגדיר את סגנון הקוד, הכלים המועדפים, ומה לא לעשות. במקום להגיד כל פעם "use TypeScript, no semicolons, Tailwind only", תכתבו את זה פעם אחת ב-CLAUDE.md ו-Claude Code יזכור.

רוצים לנסות בעצמכם?

פתחו את Claude Code ותתחילו לתרגל. ההבדל בין פרומפט בינוני למצוין הוא רק עניין של תרגול.
